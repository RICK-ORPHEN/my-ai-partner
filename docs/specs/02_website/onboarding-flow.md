# オンボーディングフロー / Onboarding Flow

## 1. 2種類のオンボーディング
| ページ | 役割 | 想定時間 |
|---|---|---|
| onboarding.html | 軽量版。業種＋目的だけ取得して簡易ルートを返す | 90秒 |
| interactive-onboarding.html | 本命。業種×目的×レベル×制約から完全ルート生成 | 5〜8分 |

両者は独立ではなく、**軽量版 → 本命版へアップグレード可能** な導線にする。

## 2. interactive-onboarding.html の入力項目

### Step 1 — 業種
- 飲食 / 小売 / 不動産 / 医療 / 士業 / 建設・製造 / 美容 / 教育 / その他

### Step 2 — 目的（作りたい実務システム）
- 社内サポートチャット
- 外部向け問い合わせチャット
- 企画書制作支援
- マニュアル整備
- 事業計画支援
- 営業管理システム
- Webサイト制作支援
- プロダクト開発支援
- LINE bot
- シフト調整 bot
- その他（自由入力）

### Step 3 — 現在のレベル
- 初心者（AIツールを触ったことがない〜少し触った程度）
- 中級（業務で使ったことがある）
- 上級（簡単な自動化/アプリを作ったことがある）

### Step 4 — 制約条件（複数選択可）
- ノーコード中心
- AI活用中心
- 外注併用
- 自社内開発
- 短期間（1ヶ月以内）
- 現場導入重視

  "estimatedPeriod": "4週間",
  "cautions": ["誤案内防止の文言設計", "営業時間外対応"]

## 4. 画面遷移
```
[Step 1 業種] → [Step 2 目的] → [Step 3 レベル] → [Step 4 制約]
  → [生成中アニメーション]
  → [結果画面: ゴール・フェーズ・最初の3レッスン・CTA]
  → [dashboard-v2.html へ]
```

## 5. 技術要件
- 現状は静的 HTML + JS で完結させる
- 結果生成は当面 **クライアント側のルール表** で実装（Phase 2 で GPT-5.4 API に差し替え）
- 入力値は localStorage に保存し、dashboard-v2 で再利用
- 離脱時も Step 進捗を保持、次回復帰時に続きから
