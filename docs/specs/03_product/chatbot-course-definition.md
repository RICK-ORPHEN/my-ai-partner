# チャットbotシステム開発コース / Chatbot Course Definition

## 1. コース概要
- コース名: チャットbotシステム開発コース
- 到達目標: 受講者が、自社または顧客向けのチャットbotシステムを企画・設計・実装・改善できる状態になる
- 想定期間: 4〜6週間
- 成果物: 稼働するMVP bot 1本 + 運用設計書

## 2. 対象ユーザー
- 業種問わず、現場で「同じ質問対応」に時間を取られている担当者
- 社員がAIを触れる状態にしたい経営者
- LINE / Web / 社内ツールで bot を作りたい非エンジニア

## 3. 7レッスン構成

### Lesson 1 — 課題の定義
- Goal: botが解くべき課題を1文で言える
- Question: 人がやる必要のない質問対応は何か？
- Deliverable: 対象範囲定義書（対象/対象外/導入効果仮説）
- Evaluation: 対象の絞り込み度 / 役割分担の明確さ / 定量仮説

### Lesson 2 — 会話設計
- Goal: 典型的な会話フローを描ける
- Question: 最も多い質問トップ10と、それぞれの回答意図は？
- Deliverable: 会話フロー図 + NG応答リスト
- Evaluation: 網羅性 / 分岐の妥当性 / エスカレーション設計

### Lesson 3 — 情報設計
- Goal: botが参照するデータ構造を決められる
- Question: FAQで済むのか、顧客/業務データが必要か？
- Deliverable: 参照データ一覧（形式・更新頻度・権限）
- Evaluation: データ粒度 / 更新運用の現実性 / セキュリティ観点

### Lesson 4 — システム設計
- Goal: UI/API/DB/管理画面/通知/ログの全体像を描ける
- Question: 誰がどこから使い、誰が運用するか？
- Deliverable: システム構成図 + 画面一覧
- Evaluation: 運用負荷 / 拡張性 / コスト

### Lesson 5 — 最小実装（MVP）
- Goal: 1画面 / 1用途 / 1導線で動く bot を作れる
- Question: 最小で価値が成立する経路は何か？
- Deliverable: 稼働するMVP（Cowork/Claude/Gemini/ノーコード問わず）
- Evaluation: 動作 / 最小性 / ユーザー到達率

### Lesson 6 — 評価と改善
- Goal: 改善サイクルを回せる
- Question: ユーザーはどこで詰まるか？
- Deliverable: ログ分析レポート + 改善案3つ
- Evaluation: 発見の質 / 優先度判断 / 再現性

### Lesson 7 — 実務導入
- Goal: 現場に導入して KPI を見られる
- Question: 運用者は誰で、何を見続けるか？
- Deliverable: 運用設計書（担当/KPI/アラート/改善サイクル）
- Evaluation: 運用の持続性 / KPIの妥当性 / 現場適合度

## 4. 業種別の差分ポイント（抜粋）
| 業種 | 特有の論点 |
|---|---|
| 飲食 | 営業時間 / 予約 / 繁忙帯 |
| 小売 | 在庫 / 返品 / 来店誘導 |
| 不動産 | 個人情報 / 内見導線 |
| 医療 | 誤案内防止 / 受診誘導 |
| 士業 | 守秘義務 / 初回相談導線 |
| 建設・製造 | 現場 / シフト / 安全 |
| 美容 | 予約 / 指名 / カルーセル訴求 |
| 教育 | 保護者対応 / 入学導線 |

## 5. 目的別の差分ポイント
| 目的 | 特有の論点 |
|---|---|
| 社内サポート | 社員の質問 / 社内ナレッジ参照 / 権限 |
| 外部問い合わせ | 顧客の一次質問 / 誤案内防止 / エスカレーション |
| LINE連携 | 友だち追加導線 / リッチメニュー / 通知運用 |

## 6. 完成条件（コース単位）
- 7レッスン全ての JSON が揃っている
- 各レッスンが `lesson-interactive-v2.html` で出し分けできる
- 業種差分・目的差分・レベル差分が最低3業種 × 3目的 × 3レベル分、埋まっている
- GPT-5.4 の評価プロンプトが各レッスンに紐づいている
