# 目的差分ルール / Goal Difference Rules

## 1. 目的
「同じ業種でも、社内用か顧客用かで別物になる」問題に対応するため、目的別の観点を差し込むルール表。

## 2. 目的カテゴリ
1. 社内サポートチャット
2. 外部向け問い合わせチャット
3. 企画書生成支援
4. マニュアル整備支援
5. 事業計画支援
6. 営業管理支援
7. Webサイト制作支援
8. プロダクト開発支援
9. LINE bot
10. シフト調整 bot

## 3. 各目的の差分ポイント

### 3-1. 社内サポートチャット
- 利用者: 社員
- 参照データ: 社内ナレッジ / マニュアル / 人事情報
- 評価軸: 社員の質問解決率 / 一次解決率
- リスク: 権限違反、機密情報の漏洩

### 3-2. 外部向け問い合わせチャット
- 利用者: 顧客 / 見込客
- 参照データ: FAQ / 商品情報 / 店舗情報
- 評価軸: CV率 / 電話削減率
- リスク: 誤案内、個人情報の取扱い

### 3-3. 企画書生成支援
- 利用者: 営業 / 企画
- 参照データ: 過去企画書 / テンプレート
- 評価軸: 作成時間短縮 / 初稿品質
- リスク: 情報の陳腐化、機密情報の学習流出

### 3-4. マニュアル整備支援
- 利用者: 現場 / 教育担当
- 参照データ: 既存マニュアル / 動画 / 現場メモ
- 評価軸: 改訂サイクル / 現場の参照率
- リスク: バージョン管理の破綻

### 3-5. 事業計画支援
- 利用者: 経営者 / 事業責任者
- 参照データ: 市場データ / 自社財務
- 評価軸: 意思決定スピード
- リスク: 数字の誤り、開示情報への影響

### 3-6. 営業管理支援
- 利用者: 営業 / マネージャー
- 参照データ: 顧客 / 商談 / 売上
- 評価軸: 商談化率 / 失注要因把握
- リスク: 個人情報、CRM連携の鮮度

### 3-7. Webサイト制作支援
- 利用者: 制作担当 / 経営者
- 参照データ: デザインガイドライン / ブランド資産
- 評価軸: 公開速度 / 品質
- リスク: 著作権 / アクセシビリティ

### 3-8. プロダクト開発支援
- 利用者: PM / エンジニア
- 参照データ: 要件 / 過去プロジェクト
- 評価軸: 仕様の揺れ減少
- リスク: 仕様の暗黙化

### 3-9. LINE bot
- 利用者: 顧客（LINE上）
- 参照データ: FAQ / 予約 / キャンペーン
- 評価軸: 友だち追加数 / ブロック率 / 予約CV
- リスク: 通知過多、セグメント誤配信

### 3-10. シフト調整 bot
- 利用者: 店長 / スタッフ
- 参照データ: シフト表 / 希望 / 労務ルール
- 評価軸: 調整時間削減
- リスク: 労務法違反、連続勤務

## 4. 運用ルール
- 目的を追加する場合は必ず「利用者 / 参照データ / 評価軸 / リスク」の4項を埋めること
- 目的と業種の交差は `curriculum-framework.md` の差分注入ロジックで処理する
