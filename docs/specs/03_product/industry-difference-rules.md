# 業種差分ルール / Industry Difference Rules

## 1. 目的
同一の共通レッスンに対して、業種別の観点を差し込むためのルール表。
`curriculum-framework.md` の `industry_diff` フィールドに投入される。

## 2. 共通の観点（全業種に適用）
- 顧客/利用者の一次質問は何か
- 人がやるべき部分とbotがやるべき部分
- 誤案内/誤提供のリスク
- 緊急時のエスカレーション

## 3. 業種別ルール

### 3-1. 飲食
- 一次質問: 営業時間 / 予約可否 / アクセス / メニュー / アレルギー
- NG: アレルギー回答をbotだけで完結させる
- 導入効果KPI: 電話問い合わせ数削減 / 予約転換率
- 特有リスク: 営業時間外の誤回答、季節メニューの情報陳腐化

### 3-2. 小売
- 一次質問: 在庫 / サイズ / 返品 / 店舗在庫 / 来店時間
- NG: 実在庫と乖離した案内
- 導入効果KPI: CS問い合わせ削減 / 再来店率
- 特有リスク: 在庫API連携の鮮度

### 3-3. 不動産
- 一次質問: 物件有無 / 内見 / 家賃 / 条件 / 初期費用
- NG: 個人情報を最初のターンで要求する
- 導入効果KPI: 内見予約数 / 問い合わせ〜内見転換率
- 特有リスク: 個人情報保護、宅建業法の表示義務

### 3-4. 医療
- 一次質問: 診療時間 / 予約 / 症状相談 / アクセス
- NG: 症状に対する医学的判断を返す
- 導入効果KPI: 受付電話数削減 / 新患比率
- 特有リスク: 医療広告ガイドライン、誤案内の健康リスク

### 3-5. 士業
- 一次質問: 初回相談の可否 / 費用 / 対応領域 / 実績
- NG: 個別案件の法的判断を返す
- 導入効果KPI: 初回相談予約数
- 特有リスク: 守秘義務、弁護士法72条等への抵触

### 3-6. 建設・製造
- 一次質問: 工期 / 見積 / 対応エリア / 安全 / 実績
- NG: 現場条件抜きの確定見積を返す
- 導入効果KPI: 有効見積依頼数
- 特有リスク: 現場ごとの条件差が大きい

### 3-7. 美容
- 一次質問: 予約 / 指名 / 施術時間 / 料金 / キャンセル
- NG: スタッフ指名の無断割当
- 導入効果KPI: 予約数 / 指名率
- 特有リスク: 施術内容の誤案内、返金対応

### 3-8. 教育
- 一次質問: 入学条件 / 授業料 / 体験授業 / 送迎
- NG: 保護者の個人情報を対話冒頭で要求
- 導入効果KPI: 体験授業予約数 / 入会率
- 特有リスク: 未成年の個人情報保護

## 4. 運用ルール
- 新業種を追加する場合は、本書の節を増やしてから JSON 側に反映する
- 業種ルールの変更は `admin-curriculum-rules.html` 経由で履歴を残す（将来）
