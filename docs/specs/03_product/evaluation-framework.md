# 評価フレームワーク / Evaluation Framework

## 1. 目的
GPT-5.4 による提出物評価を、一貫性・再現性・受講者の成長の観点で設計する。

## 2. 評価の3階層
1. **Surface** — 形式要件（文字数 / セクション / 形式）
2. **Structure** — 論理構造（目的・対象・制約・効果）
3. **Insight** — 気づき（現場理解 / リスク認知 / 新しい視点）

## 3. 共通評価軸（全レッスン共通）
| 軸 | 説明 | 配点 |
|---|---|---|
| 明確性 | 対象と範囲が1文で言えるか | 20 |
| 妥当性 | 根拠がデータ/現場観察に基づくか | 20 |
| 実現可能性 | 現場で実装できる現実性があるか | 20 |
| リスク認識 | 誤案内/漏洩/法令リスクに触れているか | 20 |
| 発展性 | 次のレッスンにつながる論点を含むか | 20 |

## 4. スコアと帰結
| 合計 | 判定 | 次アクション |
|---|---|---|
| 80以上 | Pass | 次レッスンへ進む |
| 60〜79 | Revise | ヒントをもう1段深くして再提出 |
| 〜59 | Retry | 問いを別角度から出し直す |

## 5. 出力フォーマット（GPT-5.4 → 受講者）
```json
{
  "score": 74,
  "level": "Revise",
  "by_axis": {
    "clarity": 18,
    "validity": 14,
    "feasibility": 16,
    "risk_awareness": 12,
    "extension": 14
  },
  "strengths": [
    "対象範囲の絞り込みは良い"
  ],
  "missing_views": [
    "誤案内リスクへの言及がない",
    "定量仮説が主観的"
  ],
  "next_questions": [
    "この誤案内が発生したとき、現場は誰がどう止めますか？"
  ]
}
```

## 6. 禁止事項（評価エンジンに対する原則）
- 完成答えを返さない
- 「正解」という言葉を使わない
- 個人を否定する表現を返さない
- 提出物の書き直し全文を返さない
