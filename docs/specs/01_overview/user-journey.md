# ユーザージャーニー / User Journey

## 1. 全体像
LP流入 → オンボーディング → カリキュラム生成 → 学習 → 提出 → AI評価 → 次レッスン → 到達（実務システム稼働）

## 2. 各フェーズ詳細

### Phase A — 流入 / 興味形成
- 入口: index.html（業種別導線） / corporate.html / individual.html
- ユーザーが感じるべきこと: 「自分の業種のために作られている」「答えを渡さずに考えさせる」
- CTA: 「無料で自分専用カリキュラムを見る」→ interactive-onboarding.html

### Phase B — 導入 / 現在地診断
- ページ: interactive-onboarding.html
- 取得する入力:
  1. 業種（8カテゴリ＋その他）
  2. 目的（作りたい実務システム）
  3. 現在のレベル
  4. 制約（ノーコード / 内製 / 期間）
- 出力: 推奨ゴール、到達フェーズ、最初の3レッスン、想定成果物、難易度、所要期間

### Phase C — 学習開始
- ページ: dashboard-v2.html → lesson-interactive-v2.html
- 体験:
  1. レッスン冒頭で「問い」が提示される
  2. すぐに答えは出ない
  3. ヒントを段階的にリクエスト可能（Mode A→D）
  4. Cowork / Claude / Gemini で実装
  5. 提出
  6. GPT-5.4 が評価
  7. 足りない視点・次の論点が返る
  8. 次レッスンへ

### Phase D — 継続 / 可視化
- ページ: dashboard-v2.html, dashboard-skills-v2.html
- 可視化対象: 進捗、獲得スキル、次の課題、提出履歴、AI評価サマリ

### Phase E — 到達 / 実務稼働
- 受講者が以下のいずれかを自力で稼働させている状態:
  - 社内サポートチャット
  - 外部向け問い合わせチャット
  - 企画書生成支援
  - マニュアル整備支援
  - 事業計画支援
  - 営業管理支援
  - Webサイト制作支援
  - プロダクト開発支援
  - LINE bot
  - シフト調整 bot

### Phase F — マッチング / 人材DB
- ページ: dashboard-matching-v2.html, talent.html
- 到達者を AI 人材として登録、企業と接続（将来のマッチング手数料収益の源泉）

## 3. 成功指標（ジャーニーKPI）
| フェーズ | 指標 | 目標 |
|---|---|---|
| A | LP→オンボーディング遷移率 | 15% |
| B | オンボーディング完了率 | 70% |
| C | 初回レッスン着手率 | 60% |
| D | 4週間継続率 | 40% |
| E | 実務稼働到達率 | 25% |
| F | 人材DB登録率 | 15% |
