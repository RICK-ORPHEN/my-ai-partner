## 1. このサービスの正体（v2）

**業務課題を自分で解ける思考力をつける、飲食業特化のAI実践道場。**

Cowork / Claude / ChatGPT / Gemini は実践環境として使うが、売っているのはツールの操作方法ではない。売っているのは:
- 現場の課題を分解する力
- AIに何を任せ、何を人がやるかを判断する力
- 最小限のシステムを自分で設計・実装・改善する力

ツールが変わっても思考力は残る。これが競合との本質的な差。

---

## 2. 初期ターゲット: 飲食業に絞る理由

1. **既に作ってある**: lesson-interactive-v2.html が飲食×予約で構築済み
2. **意思決定が速い**: オーナー1人で即決できる。法人稟議が要らない
3. **効果が即日測定可能**: 「電話が減った」「予約が増えた」が1週間で出る
4. **店舗数が多い**: 日本の飲食店 約67万店。ターゲット母数が巨大
5. **課題が明確**: 予約、問い合わせ、シフト、メニュー、レビュー対応

LP上では8業種を「対応中」として見せるが、**営業・広告・教材の深掘りは飲食のみ**。

## 5. 3本柱の再定義と依存関係

```
柱1: AI実践道場（飲食特化）
  ↓ 修了者が出る
柱2: ポートフォリオ / スキル可視化
  ↓ 修了者100名+企業20社
柱3: マッチング（作らない。引き合いが来てから判断）
```

### 柱1 — AI実践道場（最優先・これだけで課金する）
- 飲食業に特化した chatbot コース 7 レッスン
- 問い → ヒント → 実装 → AI評価 のループ
- 成果物: 現場で稼働する MVP bot

### 柱2 — ポートフォリオ（柱1 が回ってから）
- 修了者のスキルスコアと成果物を公開URL化
- PDF認定証の出力
- **プラットフォームではなく「個人のURL」として提供**

### 柱3 — マッチング（当面は手動）
- プラットフォームは作らない
- 修了者100名を超えたら、企業からの引き合いをメールで仲介
- 仕組み化は需要が見えてから

## 6. 無料体験の出口設計（課金転換の仕掛け）

```
[Lesson 2 完了]

  → CTA: 「続きを始める → ¥49,800/月」

```

### 設計の意図
- Lesson 1〜2 で「考える力がついた」実感を渡す
- Lesson 3 以降の「作る力」に課金する
- 答えを渡さないAIの設計思想がここで活きる
- 「ここまではできた。でもここから先が本番」と体感させる

## 7. 飲食特化コースの再定義

### コース名
**「飲食店AI活用 実践コース — 予約bot から始める業務改革」**

### 7レッスンの飲食版
| # | タイトル | 飲食特化の問い | 成果物 |
|---|---|---|---|
| 1 | 課題の定義 | 1日の電話のうち、botに任せていい問い合わせは何件？ | 対象範囲定義書 |
| 2 | 会話設計 | 「今日空いてますか？」への返し方は何パターン？ | 会話フロー図 |
| 3 | 情報設計 | 予約台帳・メニュー・営業時間のどれをbotに渡す？ | 参照データ設計書 |
| 4 | システム設計 | LINE / Web / 電話、どこで受けてどこで管理する？ | 構成図 |
| 5 | 最小実装 | 「予約の空き確認」だけで動くbotを1つ作れ | 稼働するMVP |
| 6 | 評価と改善 | 最初の1週間で客はどこで詰まった？ | 改善レポート |
| 7 | 実務導入 | 誰が毎日見て、何の数字を追うか？ | 運用設計書 |

### 無料区間（Lesson 1-2）で渡す価値
- Lesson 1: 「うちの店で本当にbotが要る場面」が整理される → 経営者に見せられる
- Lesson 2: 「お客さんの質問パターンと応答方針」が図になる → 実務資料として使える

**無料でもらえる2つの資料だけで「やる価値がある」と思わせる**。これが課金転換の核。
