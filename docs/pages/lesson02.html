Part 0 — What is API?

APIとは？

まず最初に、API（エーピーアイ）とは何かを理解しましょう。
プログラミングの経験がなくても大丈夫です。

APIを「コンセント」で理解する

🔌
APIは「コンセント」のようなものです。
家電（あなたのアプリ）をコンセント（API）に差し込むと、電力（AIの能力）が使えます。
電力会社の内部の仕組みを知らなくても、コンセントに繋げば電気が来る。APIも同じです。

図で見てみよう

あなたのアプリ
（Webサイト、ツール等）
リクエスト送信 →
API
（窓口・コンセント）
処理を依頼 →
AI モデル
（GPT, Claude等）
結果を受け取る
（テキスト、画像等）
← レスポンス返却
API
（結果を中継）
← 結果を生成
AI が処理
（文章生成、分析等）

ポイント

Point 01
APIはプログラムとプログラムをつなぐ「窓口」
Point 02
リクエスト（お願い）を送ると、レスポンス（結果）が返ってくる
Point 03
AIの内部の仕組みを知らなくても使える
Point 04
APIキー（鍵）があれば、誰でもAIの能力にアクセスできる

Part 2 — Hello World

簡単な呼び出し体験

APIを実際に「触って」みましょう。
下のサンプルコードは、「文章を送ったら、AIが返事をくれる」という最もシンプルな例です。

Python - Hello World API Call sample.py
import openai # APIキーを設定（本番では環境変数に） client = openai.OpenAI(api_key="your-api-key") # AIに質問を送る response = client.chat.completions.create( model="gpt-4", messages=[ {"role": "user", "content": "こんにちは！自己紹介してください。"} ] ) # 結果を表示 print(response.choices[0].message.content)

上のコードの流れ：

APIキー設定
→
メッセージ作成
→
API送信
→
結果受け取り

試してみよう（シミュレーション）

下の入力欄にメッセージを入れて「実行」を押すと、APIの動きをシミュレーションで体験できます。

Part 4 — Your Design

API呼び出し設計の作成

ヒントを参考に、入力 → 処理 → 出力の流れを擬似コードで表現してみましょう。
完璧でなくてOK。日本語混じりでもOKです。

例：こんな感じで書いてみよう example
# 入力：お客様からのメール文 # 出力：返信メールの下書き 1. メール本文を受け取る 2. APIに送信（役割：カスタマーサポート担当） 3. 返信文を受け取る 4. 下書きとして保存 5. 担当者が確認して送信

※ ヒントを見返したい場合は「戻る」ボタンでPart 3を確認できます。

Part 6 — Build Output

成果物を作成

最後に、あなたの設計をもとに実際にAPIを呼び出すコードを体験しましょう。
下のサンプルをベースに、自分の用途に合わせて書き換えてみてください。

JavaScript - API Call Template app.js
// あなたの設計を元にしたAPI呼び出しテンプレート async function callAI(userInput) { const response = await fetch("https://api.openai.com/v1/chat/completions", { method: "POST", headers: { "Content-Type": "application/json", "Authorization": `Bearer ${API_KEY}` // 環境変数から }, body: JSON.stringify({ model: "gpt-4", messages: [ { role: "system", content: "あなたは○○の専門家です" }, { role: "user", content: userInput } ] }) }); const data = await response.json(); return data.choices[0].message.content; }
