# lp-03 — エネルギッシュ・ベンチャー / アーバン・プレミアム

スタートアップ・ベンチャー企業・ファッション・エネルギーブランド向け。
「勢い・洗練・プロフェッショナリズム・情熱」が同居するデザイン言語。

参考：スポーツブランド、ハイプ系スタートアップ、アーバンファッション、クリエイティブエージェンシー

---

## lp-01・lp-02との根本的な違い

| | lp-01（デジタル・夜） | lp-02（ボールド・昼） | lp-03（エネルギー・炎） |
|---|---|---|---|
| 背景の主役 | 黒＋グラデーション | ライトグレー＋幾何学 | **オレンジレッド＋黒＋白** |
| アクセント | 電気ブルー | 蛍光イエロー | **バーントオレンジ** |
| 構造の作り方 | カード・角丸 | シェイプ・面 | **横線（ルール）** |
| 装飾 | オーロラグラデーション | 幾何学シェイプ | **グラデーショングロー** |
| 雰囲気 | 冷静・知的 | エネルギー・遊び | **情熱・スピード・本気** |

**このスタイルの核心：横線（thin rule）が視覚的秩序を作り、オレンジレッドが感情を燃やす。**

## 1. カラーシステム

### ベースパレット（3+1色）

```
オレンジレッド（主役）: #D93B00  または  #E03A00
黒:                    #0A0A0A
白:                    #FFFFFF
グロー（テクスチャ用）: オレンジ→黒のグラデーション（後述）
```

### 背景の切り替えパターン（このスタイルの命）

```
表紙・最終スライド → オレンジレッド + 黒グロー（グラデーションテクスチャ）
章扉（セクション頭） → フル・オレンジレッド（テクスチャなし・フラット）
コンテンツ（暗）     → 黒背景（写真・ポートフォリオ強調）
コンテンツ（明）     → 白背景（テキスト・データ主体）
スプリット（2分割）  → 左：写真フル、右：白背景テキスト
```

**原則：オレンジ → 黒 → オレンジ → 白 → 黒 の交互リズムが理想。**
白の連続は2枚まで。オレンジは「章の区切り」として機能させる。

### グラデーションテクスチャ（表紙・エンドスライド専用）

```css
/* オレンジ基調・黒がにじみ出る */
background:
  radial-gradient(ellipse at 20% 50%, rgba(0,0,0,0.85) 0%, transparent 50%),
  radial-gradient(ellipse at 80% 20%, rgba(0,0,0,0.6) 0%, transparent 45%),
  #D93B00;

/* さらにノイズテクスチャを薄く重ねる場合 */
/* SVG filter or CSS noise: opacity 0.04〜0.06 */
```

白グロー版（エンド用）：

```css
background:
  radial-gradient(ellipse at 80% 60%, rgba(255,220,180,0.7) 0%, transparent 50%),
  radial-gradient(ellipse at 10% 80%, rgba(0,0,0,0.8) 0%, transparent 45%),
  #D93B00;
```

### オレンジレッドの使い方（白背景スライドで）

- 章番号（01, 02...）の色
- ↗ 矢印アイコンの色
- サブ見出しのアクセント
- **使う量：1スライドあたり10〜15%以下**

## 3. 横線（thin rule）システム — このスタイルの最大の特徴

このデザインは横線で「秩序と格調」を作る。装飾ではなくレイアウトの骨格。

### 線のスペック

```css
/* 標準ルール */
border-top: 1px solid rgba(255,255,255,0.3);  /* ダーク背景上 */
border-top: 1px solid rgba(0,0,0,0.15);       /* 白背景上 */

/* セクション区切りルール（全幅）*/
width: 100%;
height: 1px;
background: 上記と同様;
margin: 0;  /* パディングなし・エッジまで伸ばす */
```

### ルールの配置パターン

```
┌─────────────────────────────────────────────────┐
│ [ラベル左]                     [↗ アイコン右]   │
├─────────────────────────────────────────────────┤  ← 上ルール
│                                                 │
│  コンテンツエリア                               │
│                                                 │
├─────────────────────────────────────────────────┤  ← 下ルール
│ [日付・URL左]                   [右テキスト]    │
└─────────────────────────────────────────────────┘
```

**全スライドに上下ルール + ヘッダー情報（ラベル・ページ番号）を統一配置する。**
これがこのスタイルの「フォーマット感」と「プロ感」の源泉。

## 4. ↗ アロー・アイコン（繰り返しモチーフ）

このデザインの統一モチーフは斜め右上向き矢印 ↗。

```css
/* 全スライド右上に配置 */
.nav-arrow {
  position: absolute;
  top: 16px;
  right: 24px;
  font-size: 20px;
  color: rgba(255,255,255,0.7);  /* ダーク背景 */
  /* または */
  color: #D93B00;  /* 白背景 */
}

/* 章番号横の小矢印（白背景スライド）*/
.section-arrow {
  color: #D93B00;
  font-size: 18px;
  margin-right: 12px;
}
```

矢印の使い場所：
- 全スライド右上（ナビゲーション感）
- 白背景スライドの見出し左（↗ Goal 01）
- CTAの前（→ お問い合わせ）

## 8. 写真・ビジュアルの扱い

```css
/* 写真は必ず角丸なし */
border-radius: 0;

/* カラー写真はそのまま使う（色加工しない）*/
/* ただし写真の雰囲気はシネマティック・ハイコントラストが合う */

/* 写真を並べる時は隙間 2〜4px */
gap: 2px;

/* スプリット写真は画面の半分を占領する */
width: 50%;
height: 100vh;  /* or 100% */
object-fit: cover;
```

## 10. NGルール

| NG | 理由 |
|---|---|
| 横線を省く | このスタイルの骨格が消える |
| ↗ アイコンを使わない | 統一感と「方向性・勢い」の表現が失われる |
| 角丸カードを使う | 柔らかさが出てしまい「本気感」が薄れる |
| 本文を太くする | 見出しとのコントラストが崩れる |
| オレンジを白背景の塗りに多用する | アクセントの力が薄れる |
| 幾何学シェイプを追加する | lp-02になってしまう |
| 3色以上のカラーを使う | 散漫になる |
| フォントを細くしすぎる（Thin）| 勢いが消える。Light止まり |
